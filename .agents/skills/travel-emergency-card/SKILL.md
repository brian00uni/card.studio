# Travel Emergency Card Skill

## 목표

`topics.csv`의 다음 대기 도시 하나를 골라, 최신 공식 정보와 실제 약품 자산을 이용한 HTML 카드 7장을 만든다.

## 실행 순서

1. `AGENTS.md`와 `DESIGN.md`를 읽는다.
2. `topics.csv`에서 상태가 `대기`인 항목 중 순서가 가장 빠른 한 행을 선택한다.
3. 대상 국가의 `assets/medicine/<country>/`와 `medicine-assets.csv`를 확인한다.
4. 공식 출처를 중심으로 최신 정보를 조사한다.
5. 조사 결과를 `data/cities/<slug>.json`에 구조화한다.
6. `node scripts/validate-data.mjs <json-path>`를 실행한다.
7. 오류가 있으면 임의로 채우지 않고 `qa-report.md`에 기록한다.
8. 데이터가 유효하면 `node scripts/build-html.mjs <json-path> <output-dir>`를 실행한다.
9. 생성된 HTML 7장을 열어 overflow, 글자 잘림, 계정명 표기를 확인한다.
10. 필요하면 `npm run render -- <output-dir>`로 PNG를 만든다.
11. `sources.md`, `caption.txt`, `text-version.txt`, `qa-report.md`를 완성한다.
12. 모든 완료 조건을 통과한 경우에만 `topics.csv` 상태를 `완료`로 변경한다.

## 조사 우선순위

1. 대한민국 외교부와 해당 대한민국 공관
2. 현지 정부·관광청·긴급기관
3. 병원 공식 웹사이트
4. 의약품 제조사 공식 웹사이트
5. 공식 한인기관
6. 대표 커뮤니티는 보조 정보로만 사용

## 카드 데이터 핵심

### 1장

도시명과 위급상황 용도를 즉시 공개한다.

### 2장

한국어 지원 긴급번호를 첫 번째로 공개한다. 번호를 가장 크게 빨강으로 표시한다.

### 3장

지원 언어가 확인된 병원만 사용한다. 운영시간·휴진·전화·주소를 최소 정보로 정리한다.

### 4장

공식기관과 대표 커뮤니티를 구분한다. 제목에 `검색어`를 쓰지 않는다.

### 5장

호텔, 택시, 병원 접수처에 그대로 보여줄 짧은 현지어 문장을 제공한다.

### 6장

실제 약품 사진만 사용한다. 승인 자산이 없으면 `NEEDS_ASSET.md`를 만들고 중단한다.

### 7장

한국어 긴급번호부터 다시 요약한다.

## 후킹

표지 문구 후보 3개를 내부 생성하되 사용자 선택을 기다리지 않는다. 아래 기준 최고점 하나를 선택한다.

- 즉시 이해
- 과장 없는 위급성
- 저장 이유
- 도시명 인지
- 짧은 문장

## 출력 품질

- 1080×1920 HTML
- 로컬 Pretendard
- 외부 폰트 요청 0건
- 상단 오른쪽 `@gaseongbi_crew`
- 문단형 금지
- 전화번호는 실제 텍스트
- 이미지 생성 모델로 본문 생성 금지

## 실패 처리

다음이면 완료하지 않는다.

- 공식 출처 확인 실패
- 전화번호 충돌
- 지원 언어 불명확
- 현지어 검수 실패
- 실제 약품 이미지 없음
- HTML 7장 중 일부 누락
- 글자 잘림 또는 overflow
