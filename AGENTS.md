# AGENTS.md

## 프로젝트 목적

한국인 해외여행자가 현지에서 갑자기 아프거나 다쳤을 때 바로 꺼내 사용할 수 있는 도시별 위급정보 카드 7장을 제작한다.

우선순위는 다음과 같다.

1. 정보 정확성
2. 긴급 상황에서의 즉시 사용성
3. 모바일 가독성
4. 실제 제품 식별 가능성
5. 디자인 일관성
6. 저장 가치

긴급전화나 핵심 정보를 다음 장을 보게 만들기 위한 미끼로 숨기지 않는다.

## 필수 출력

각 실행은 `output/YYYY-MM-DD_도시명/` 폴더에 아래 파일을 만든다.

- `card-01-cover.html`
- `card-02-korean-phone.html`
- `card-03-hospital.html`
- `card-04-community.html`
- `card-05-show-screen.html`
- `card-06-medicine.html`
- `card-07-summary.html`
- `card-data.json`
- `caption.txt`
- `text-version.txt`
- `sources.md`
- `qa-report.md`

PNG 렌더링 환경이 준비되어 있으면 같은 이름의 PNG도 생성한다. PNG 실패가 HTML 생성 실패로 이어지지 않게 한다.

## 글꼴

- 기본 글꼴은 로컬 `assets/fonts/PretendardVariable.woff2`다.
- Google Fonts를 사용하지 않는다.
- 외부 폰트 CDN을 사용하지 않는다.
- 임의의 명조체, 붓글씨체, 장식 서체를 섞지 않는다.
- 일본어 글리프가 부족할 때만 로컬 시스템 일본어 산세리프 폰트를 보조 폴백으로 사용한다.

## 정보 조사

- 외교부, 현지 대한민국 공관, 현지 정부·관광청, 병원 공식 사이트, 제조사 공식 사이트를 우선한다.
- 블로그·카페·검색 결과 요약문만으로 전화번호, 운영시간, 진료 언어를 확정하지 않는다.
- 전화번호, 주소, 운영시간, 지원 언어, 확인 날짜를 기록한다.
- 정보가 충돌하면 임의로 선택하지 말고 `qa-report.md`에 기록한다.
- 확인되지 않은 사실은 카드에서 제외한다.
- 대표 커뮤니티는 공식기관과 명확히 구분한다.

## 의약품

- 실제 제품 패키지 사진만 사용한다.
- 이미지 속 현지어 제품명이 선명해야 한다.
- 승인된 사진이 없으면 유사 패키지를 생성하지 않는다.
- 사진이 없으면 `NEEDS_ASSET.md`를 만들고 카드 6을 보류한다.
- 어린이에게 성인용 의약품을 임의로 권하지 않는다.
- 제품보다 증상 설명과 약사 확인을 우선한다.

## 디자인

- `DESIGN.md`를 반드시 따른다.
- 도시 실사 배경을 사용하지 않는다.
- 국기 자체를 크게 그리지 않는다.
- 국가색은 면, 선, 작은 기하 도형으로만 사용한다.
- 모든 카드 상단 오른쪽에 `@gaseongbi_crew`만 넣는다.
- `가성비 크루`, `김대장`, 다른 계정명은 카드에 넣지 않는다.
- 한 카드의 가장 중요한 정보는 하나만 크게 강조한다.
- 장식보다 전화번호와 행동 문장의 가독성을 우선한다.

## 카드 순서

1. 표지
2. 한국어가 되는 긴급전화
3. 병원
4. 공식 한인기관과 대표 카페·커뮤니티
5. 말이 안 통할 때 보여주는 현지어 문장
6. 실제 패키지 사진이 포함된 약품
7. 한눈 요약

## 금지

- AI 이미지 안에 전화번호와 본문을 생성
- 긴급전화 공개를 뒤로 미루기
- 가짜 후기
- 확인 없이 `24시간`, `한국어 가능` 표기
- AI가 만든 가짜 약품 패키지
- 도시 실사 배경
- 과도한 그림자·금색 테두리·리본·별·왕관
- Google Fonts 또는 외부 폰트 CDN
- 긴 문단과 과한 이모지
